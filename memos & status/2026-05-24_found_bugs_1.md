# Баги в пайплайне LLM evaluation с reasoning моделями

## EVAL SETUP

```python
@task
def test_eval():
    return Task(
        dataset =example_dataset("theory_of_mind"),
        solver=[
            chain_of_thought(),
            generate(),
            self_critique()
        ],
        scorer=model_graded_fact()
    )
```

## Баг 1: Пустой text output при self_critique (reasoning модель)

### Что нашлось
При использовании солвера `self_critique()` с reasoning моделью (`deepseek-v4-flash` через API провайдера) второй вызов `generate()` возвращает пустой `text` при непустом `reasoning` блоке:

```json
{
  "type": "reasoning",
  "reasoning": "The original answer is correct, so I'll repeat it exactly.\n\nANSWER: crate"
},
{
  "type": "text",
  "text": ""   // пустой
}
```

### Причина
Когда `self_critique` говорит "original answer is fully correct", reasoning модель считает задачу выполненной и помещает ответ внутрь `<think>` блока, не генерируя текстовый output. Архитектурный конфликт между iterative refinement солвером и hybrid think/non-think моделями.

### Затронутые модели
Потенциально любая reasoning модель с явным `<think>...</think>` разделением: DeepSeek-R1, DeepSeek-V3.1/V3.2, DeepSeek-V4-Flash, Qwen3.

### Статус воспроизведения
- ✅ Подтверждён на `deepseek-v4-flash` через API (лог есть)
- ❌ Не воспроизведён на `deepseek-r1-distill-qwen-7b-awq` локально (150 вопросов, 0 случаев)

### Гипотезы
- Баг специфичен для крупных hybrid think/non-think моделей
- На дистиллятах поведение может отличаться из-за упрощённой логики reasoning

### Необходимые доп. тесты
- [ ] Прогнать тот же датасет с `self_critique()` через DeepSeek API (`deepseek-reasoner`) для изоляции — модель vs провайдер
- [ ] Проверить зависимость от размера: `deepseek-r1-distill-qwen-14b` или `llama-8b`
- [ ] Минимальный репро без Inspect AI — прямой API вызов с self_critique промптом

### Предлагаемый фикс
Fallback в солвере: если `completion` пустой — извлекать `ANSWER:` из reasoning блока.

## Баг 2: Некорректный парсинг GRADE в model_graded_fact

### Что нашлось
Модель-судья (`deepseek-r1-distill-qwen-7b-awq`) стабильно нарушает формат вывода при грейдинге. Вместо `GRADE: C/I` генерирует варианты:

| Сэмпл | Что написала модель | Ожидалось |
|-------|-------------------|-----------|
| 54    | `GRAND: I`        | `GRADE: I` |
| 67    | `ANSWER: C`       | `GRADE: C` |
| 17 (пилот) | `GRID: C`  | `GRADE: C` |

Результат: `"Grade not found in model output"`, поле `answer` в UI пустое — несмотря на то что `completion` солвера непустой и правильный.

### Причина
Две независимые проблемы:

**2a. Качество модели-судьи** — малая reasoning модель (7b) не следует точному формату инструкции. Большая модель (`deepseek-v4-flash`) этот баг не воспроизводит.

**2b. Логика заполнения `answer`** — `model_graded_fact` заполняет поле `answer` только при успешном парсинге `GRADE:`. При падении парсинга `answer` остаётся пустым, хотя `completion` есть и корректен. Ответ модели и оценка ответа концептуально независимы, но в коде связаны.

### Статус воспроизведения
- ✅ Стабильно воспроизводится на 7b как judge (3 случая на ~180 вопросах)
- ❌ Не воспроизводится на большой модели

### Гипотезы
- Баг 2a решается промптингом судьи или robust regex парсингом
- Баг 2b может быть осознанным дизайн-решением Inspect AI — **требует проверки исходников** `model_graded_fact` перед формулировкой как баг

### Необходимые доп. тесты
- [ ] Проверить исходники Inspect AI: откуда берётся `answer` в объекте score — из `completion` или из judge pipeline
- [ ] Проверить помогает ли более строгий промпт судьи (explicit: "you MUST end with exactly GRADE: C or GRADE: I")
- [ ] Попробовать structured output для judge вместо свободного текста

### Предлагаемые фиксы
- **Краткосрочный:** robust regex для парсинга — `GRADE|GRAND|GRID` + fallback на `ANSWER:`
- **Долгосрочный:** если 2b подтвердится как баг — PR в Inspect AI с заполнением `answer` из `completion` независимо от грейдинга

## Общий вывод

Оба бага проявляются при использовании reasoning моделей в пайплайне и независимы друг от друга:

- **Баг 1** — в солвере, до скорера
- **Баг 2** — в скорере, независимо от солвера

Класс проблем: **reasoning модели с `<think>` архитектурой несовместимы с рядом стандартных компонентов Inspect AI** — как на уровне солверов (self_critique), так и на уровне скореров (model_graded_fact).
